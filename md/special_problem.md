# ปัญหาพิเศษ: ระบบ Sports Match

**แอปพลิเคชันจับคู่นักกีฬาและจัดการกิจกรรมกีฬาสำหรับนิสิต มหาวิทยาลัยเกษตรศาสตร์ วิทยาเขตกำแพงแสน**

---

## คำนำ
(Introduction)

ในปัจจุบันนิสิตมหาวิทยาลัยเกษตรศาสตร์ วิทยาเขตกำแพงแสน จำนวนมากมีความต้องการออกกำลังกายและเล่นกีฬาร่วมกับผู้อื่น แต่มักประสบปัญหาในการหาเพื่อนที่เล่นกีฬาชนิดเดียวกัน มีระดับฝีมือใกล้เคียงกัน และมีเวลาว่างตรงกัน การติดต่อสื่อสารเพื่อนัดหมายในปัจจุบันมักกระจัดกระจายอยู่ตามกลุ่มแชทต่าง ๆ ทำให้ขาดความเป็นระบบ ไม่สามารถค้นหาผู้เล่นใหม่ ๆ ได้สะดวก และไม่มีการตรวจสอบความปลอดภัยของผู้ใช้งาน

ผู้จัดทำปัญหาพิเศษจึงเล็งเห็นถึงความสำคัญของการพัฒนาแอปพลิเคชัน **Sports Match** ซึ่งเป็นระบบจับคู่นักกีฬาแบบ Swipe-to-Match ผสมผสานกับระบบสร้างห้องนัดเล่นกีฬา (Sport Session) และระบบจัดกิจกรรมกีฬา (Event) ไว้ในแพลตฟอร์มเดียว เพื่อให้นิสิต KU กำแพงแสนสามารถหาคนเล่นกีฬาได้ตรงกีฬา ตรงระดับ และตรงเวลา พร้อมทั้งมีระบบดูแลความปลอดภัยของผู้ใช้งานผ่านฟังก์ชัน Report และ Admin Dashboard

## วัตถุประสงค์ของการทำปัญหาพิเศษ
(Objectives)

แอปพลิเคชัน Sports Match มีวัตถุประสงค์ดังต่อไปนี้

1. เพื่อพัฒนาระบบจับคู่นักกีฬา (Swipe-to-Match) และระบบสนทนาแบบเรียลไทม์ โดยมีการจัดเก็บข้อมูลด้วยฐานข้อมูล
2. เพื่อพัฒนาระบบสร้างและจัดการห้องนัดเล่นกีฬา (Sport Session) และกิจกรรมกีฬา (Event) พร้อมระบบเช็คอินด้วย QR Code และระบบผู้ดูแลระบบ (Admin) สำหรับตรวจสอบและจัดการความปลอดภัยของผู้ใช้งาน

## การตรวจเอกสาร
(Literature Review)

ในการจัดทำแอปพลิเคชัน Sports Match ต้องใช้ความรู้และเทคโนโลยีในการพัฒนาระบบ ซึ่งประกอบด้วยเครื่องมือดังต่อไปนี้

### 1. React และ TypeScript

React เป็น JavaScript Library สำหรับสร้างส่วนติดต่อผู้ใช้ (User Interface) แบบ Component-based ทำให้สามารถแบ่งหน้าจอออกเป็นส่วนย่อย ๆ ที่นำกลับมาใช้ซ้ำได้ ช่วยให้การพัฒนาและดูแลรักษาโค้ดทำได้ง่ายขึ้น ส่วน TypeScript เป็นส่วนขยายของภาษา JavaScript ที่เพิ่มระบบตรวจสอบชนิดข้อมูล (Static Typing) ทำให้ตรวจพบข้อผิดพลาดได้ตั้งแต่ตอนเขียนโค้ด ลดข้อผิดพลาดที่อาจเกิดขึ้นตอนรันจริง เหมาะสำหรับพัฒนาระบบที่มีความซับซ้อนและมีทีมพัฒนาหลายคน
(แหล่งที่มา https://react.dev, https://www.typescriptlang.org)

### 2. Node.js และ Express

Node.js คือสภาพแวดล้อมสำหรับรันภาษา JavaScript ฝั่งเซิร์ฟเวอร์ (Server-side) ทำให้สามารถใช้ภาษาเดียวกันได้ทั้งฝั่ง Frontend และ Backend ส่วน Express เป็น Web Framework ที่ทำงานอยู่บน Node.js ใช้สำหรับสร้าง REST API ได้อย่างรวดเร็ว รองรับการทำ Routing, Middleware และการตรวจสอบสิทธิ์ผู้ใช้งาน (Authentication/Authorization) ซึ่งเหมาะสำหรับพัฒนา Backend ของระบบที่ต้องรองรับผู้ใช้งานจำนวนมาก
(แหล่งที่มา https://nodejs.org, https://expressjs.com)

### 3. PostgreSQL และ Prisma ORM

PostgreSQL เป็นระบบจัดการฐานข้อมูลเชิงสัมพันธ์ (Relational Database) แบบโอเพนซอร์ส มีความเสถียรสูงและรองรับการทำงานร่วมกับข้อมูลเชิงสัมพันธ์ที่ซับซ้อน เช่น ความสัมพันธ์ระหว่างผู้ใช้ ห้องนัดเล่นกีฬา และกิจกรรม ส่วน Prisma เป็น ORM (Object-Relational Mapping) ที่ช่วยให้นักพัฒนาสามารถออกแบบโครงสร้างฐานข้อมูล (Schema) และเขียนคำสั่งดึง/บันทึกข้อมูลได้ด้วยไวยากรณ์ TypeScript แทนการเขียน SQL โดยตรง ช่วยลดข้อผิดพลาดและเพิ่มความปลอดภัยของข้อมูล
(แหล่งที่มา https://www.postgresql.org, https://www.prisma.io)

### 4. Socket.IO

Socket.IO เป็นไลบรารีสำหรับการสื่อสารแบบเรียลไทม์ (Real-time Communication) ระหว่างฝั่ง Client และ Server ผ่านโปรโตคอล WebSocket ใช้ในการพัฒนาระบบแชทและระบบแจ้งเตือนแบบทันที เช่น การแจ้งเตือนเมื่อมีผู้จับคู่สำเร็จ หรือมีผู้เข้าร่วมห้องนัดเล่นกีฬาใหม่
(แหล่งที่มา https://socket.io)

### 5. JWT (JSON Web Token)

JWT เป็นมาตรฐานสำหรับการยืนยันตัวตนและตรวจสอบสิทธิ์ผู้ใช้งาน (Authentication) ผ่านการสร้างโทเคนที่มีการเข้ารหัสลายเซ็นดิจิทัล ทำให้ระบบสามารถตรวจสอบได้ว่าผู้ใช้งานได้รับอนุญาตให้เข้าถึงข้อมูลหรือฟังก์ชันใดได้บ้าง โดยไม่ต้องเก็บสถานะ (Session) ไว้ที่ฝั่งเซิร์ฟเวอร์
(แหล่งที่มา https://jwt.io)

### 6. Cloudinary

Cloudinary เป็นบริการจัดเก็บและจัดการรูปภาพบนคลาวด์ ใช้สำหรับอัปโหลดรูปโปรไฟล์ผู้ใช้งาน รูปปกกิจกรรม และรูปสถานที่เล่นกีฬา ช่วยลดภาระของเซิร์ฟเวอร์หลักในการจัดเก็บไฟล์ และรองรับการปรับขนาด/แปลงรูปภาพอัตโนมัติ
(แหล่งที่มา https://cloudinary.com)

### 2. วิธีการ (Methodology)

ขั้นตอนการดำเนินการโครงการนี้แบ่งการพัฒนาออกเป็น 6 ขั้นตอนหลัก คือ

**1. ศึกษาปัญหาและความต้องการของระบบ**
สำรวจความต้องการของนิสิตในการหาเพื่อนเล่นกีฬา ศึกษาพฤติกรรมการนัดหมายเล่นกีฬาในปัจจุบัน และปัญหาที่พบ เช่น การหาผู้เล่นไม่ครบ การไม่ทราบระดับฝีมือของผู้เล่นคนอื่น และการขาดช่องทางที่เป็นระบบในการจัดกิจกรรมกีฬาภายในมหาวิทยาลัย

**2. วิเคราะห์ระบบ**
จากการศึกษาความต้องการ ผู้จัดทำได้กำหนดกลุ่มผู้ใช้งานออกเป็น 3 ระดับ คือ ผู้ใช้ทั่วไป (User) ผู้จัดกิจกรรม (Event Organizer) และผู้ดูแลระบบ (Admin) พร้อมกำหนดฟังก์ชันหลักของระบบ ได้แก่ ระบบจับคู่ผู้เล่นแบบ Swipe ระบบสร้างห้องนัดเล่นกีฬา ระบบจัดกิจกรรมกีฬาพร้อมเช็คอินด้วย QR Code ระบบแชท และระบบรายงาน/บล็อกผู้ใช้งานที่ไม่เหมาะสม

**3. ออกแบบระบบ**
นำข้อมูลที่ได้จากการวิเคราะห์มาออกแบบโครงสร้างฐานข้อมูล (Entity Relationship Diagram) ออกแบบ REST API และออกแบบหน้าจอผู้ใช้งาน (UI/UX) ให้ใช้งานง่าย เข้าใจง่าย และรองรับการใช้งานทั้งบนคอมพิวเตอร์และสมาร์ทโฟน (Responsive Design)

**4. พัฒนาระบบ**
พัฒนาส่วนหน้าบ้าน (Frontend) ด้วย React และ TypeScript และพัฒนาส่วนหลังบ้าน (Backend) ด้วย Node.js และ Express พร้อมเชื่อมต่อฐานข้อมูล PostgreSQL ผ่าน Prisma ORM พัฒนาระบบแชทแบบเรียลไทม์ด้วย Socket.IO และระบบยืนยันตัวตนด้วย JWT โดยแบ่งพัฒนาเป็นโมดูลย่อยตามฟังก์ชัน เช่น โมดูลสมาชิก โมดูลจับคู่ โมดูลห้องนัดเล่นกีฬา โมดูลกิจกรรม และโมดูลผู้ดูแลระบบ

**5. ทดสอบการทำงาน สรุปผลการทำงาน**
ทดสอบระบบโดยให้นิสิตกลุ่มตัวอย่างทดลองใช้งานแอปพลิเคชันจริง ทั้งการสมัครสมาชิก การจับคู่ผู้เล่น การสร้างห้องนัดเล่นกีฬา และการเข้าร่วมกิจกรรม เมื่อพบข้อผิดพลาดหรือข้อเสนอแนะจะนำมาปรับปรุงแก้ไขระบบ ก่อนสรุปผลการทดสอบ

**6. จัดทำเอกสารรูปเล่มปัญหาพิเศษ**
รวบรวมข้อมูลจากขั้นตอนการออกแบบและพัฒนาระบบ พร้อมจัดทำคู่มือการใช้งานสำหรับผู้ใช้แต่ละบทบาท (User, Organizer, Admin) และสรุปผลการดำเนินงานเป็นรูปเล่มเอกสารปัญหาพิเศษ

## สถานที่และระยะเวลา
(Places and Duration)

**สถานที่ (Place)**

สาขาเทคโนโลยีสารสนเทศและการสื่อสาร ภาควิชาวิทยาการคำนวณและเทคโนโลยีดิจิทัล คณะศิลปศาสตร์และวิทยาศาสตร์ มหาวิทยาลัยเกษตรศาสตร์ วิทยาเขตกำแพงแสน

## ประโยชน์ที่คาดว่าจะได้รับ
(Benefits)

ประโยชน์ที่คาดว่าจะได้รับจากการพัฒนาแอปพลิเคชัน Sports Match มีดังนี้

1. นิสิตมหาวิทยาลัยเกษตรศาสตร์ วิทยาเขตกำแพงแสน สามารถหาเพื่อนเล่นกีฬาที่ตรงกับชนิดกีฬา ระดับฝีมือ และเวลาว่างของตนเองได้สะดวกและรวดเร็วยิ่งขึ้น
2. เกิดชุมชนกีฬาภายในมหาวิทยาลัยผ่านการสร้างห้องนัดเล่นกีฬา (Session) และการจัดกิจกรรมกีฬา (Event) อย่างเป็นระบบ
3. มหาวิทยาลัยมีเครื่องมือช่วยดูแลความปลอดภัยของผู้ใช้งาน ผ่านระบบรายงานพฤติกรรมไม่เหมาะสมและระบบผู้ดูแลระบบ (Admin) ที่สามารถตรวจสอบและจัดการได้อย่างมีประสิทธิภาพ
