# คำถามรอบ 3 — Sports Match

> เหลือ 6 จุดที่ยังไม่ชัดพอจะ implement ได้

---

## 1. No Show — ใครตรวจสอบหลักฐาน? (Q1.2 ยังไม่ได้ตอบ)

ติ๊ก "อื่นๆ" แต่ไม่ได้กรอก

**Q:** ใครเป็นคนตรวจหลักฐานรูปภาพที่แนบมา?
- [✅] Admin ตรวจเองทุกเคส
- [ ] ระบบอนุมัติอัตโนมัติ ถ้าอีกฝ่ายไม่โต้แย้งภายใน X วัน
- [ ] อื่นๆ: ___

---

## 2. Event Organizer — อยากได้สิทธิ์อะไรเพิ่ม? (Q2.1 ยังไม่ครบ)

ในไฟล์บอกว่า "ขอสิทธิ์ role organizer ให้เลือกมากกว่านี้อีก"

**Q:** สิทธิ์ที่อยากเพิ่มให้ Event Organizer มีอะไรบ้าง?

| สิทธิ์ | ได้ | ไม่ได้ |
|---|---|---|
| ดู Event ของคนอื่นที่ตัวเองไม่ได้สร้าง | |✅|
| ยกเลิก / ปิด Event ของคนอื่น | |✅|
| ดูรายชื่อสมาชิกทุก Event ในระบบ | |✅|
| ส่ง Announcement ให้ผู้ใช้ทุกคน |✅| |
| ดูสถิติ Event (จำนวน Join / ยกเลิก) |✅| |
| Verify / ยืนยันผู้เข้าร่วม Event |✅| |

---

## 3. Block — ใน Swipe ทำงานแบบ Mutual หรือ One-sided? (Q5.2 ยังไม่ชัด)

Q5.1 ตอบแค่เรื่อง Sport Session แต่ยังไม่ตอบเรื่อง Swipe

**Q:** ถ้า A Block B ในหน้า Swipe B ยังเห็น A ได้ไหม?
- [✅] ไม่เห็น — Block ทำงานทั้งสองทาง (Mutual)
- [ ] ยังเห็นได้ — Block ทำงานทางเดียว A มองไม่เห็น B แต่ B ยังเห็น A

---

## 4. Report จากหน้า Swipe Card ได้ไหม? (Q6.2 ยังไม่ชัด)

คำตอบบอกว่า "อาจเจอโปรไฟล์ปลอม" แต่ไม่ได้ตอบตรงๆ

**Q:** ผู้ใช้กด Report คนอื่นจากหน้า Swipe Card ได้ไหม (ก่อน Match)?
- [✅] ได้ — มีปุ่ม Report บนการ์ด
- [ ] ไม่ได้ — ต้อง Match ก่อนถึง Report ได้

---

## 5. Favorite Place — ข้อมูลสนามมาจากไหน? (Q10.1 คำตอบไม่ชัด)

คำตอบซ้ำกัน "Google Maps API + Google Maps API" ไม่แน่ใจว่าหมายความว่าอะไร

**Q:** รายชื่อสนามกีฬา KU ในระบบมาจากไหน?
- [ ] Admin เพิ่มรายชื่อสนามเองผ่าน Dashboard (Fixed List)
- [ ] ดึงจาก Google Maps API อย่างเดียว (ผู้ใช้ค้นหาชื่อสนามเอง)
- [ ] Admin เพิ่มรายชื่อตายตัว + ผู้ใช้พิมพ์เพิ่มเองได้
- [✅] อื่นๆ: Google Maps API + Admin เพิ่มรายชื่อสนามเองผ่าน Dashboard (Fixed List)

## 6. Favorite Place — จำกัดกี่สนาม? (Q10.2 ไม่ได้ใส่ตัวเลข)

ติ๊ก "จำกัด" แต่ช่องตัวเลขว่างอยู่

**A:** 5 สนาม
